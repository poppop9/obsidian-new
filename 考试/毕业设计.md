# 研究方向
<u>要求</u> ：
- 研究方向最多两个，每个方向最多 8 个汉字，以分号分隔，总数不超过 17 个汉字

<u>展望</u> ：
- 基于用户行为的个性化商品推荐与高并发营销系统 ：用户在系统中有个画像，有他的购买记录，大模型会根据他的购买记录给他推荐营销抽奖的东西
- 基于个性化定制：结合用户数据，提供个性化定制的商品或服务，适应越来越多消费者对定制化、个性化需求的趋势。 
- 用户在淘宝购买了商品，就会赠送随机积分
- 当用户登录，我们可以拿到其购买数据，此时会将数据推送到 kafka 中，系统异步处理用户画像生成请求
- 用户抽奖通知与结果反馈：Kafka，WebSocket（适合长连接，实时推送）
	- 利用消息队列进行高并发的通知推送，确保系统能够在高并发环境下，实时发送抽奖结果通知给用户
	- 采用 WebSocket 进行页面的实时抽奖结果展示，无需刷新页面，提升用户体验。
- 将黑名单用户放入布隆过滤器，减少去数据库中查询是否是黑名单用户

# 实现方式
## 开源大模型
有一个用户，他买过：苹果手机，联想电脑，充电器，充电线。
我有四个组合的产品推销给他，你帮我看看，那个组合他更有可能会消费：
A组：内裤，袜子，包包，电脑包
B组：无线充电器，耳机，音响，鞋子
C组：西瓜，哈密瓜，梨子
D组：冰箱，空调，洗衣机
你只需要回答组名就好了，其他不用多说

## Python
- **FastAPI**：用于构建 RESTful API，处理 Kafka 消息。
- **Confluent Kafka Python**：与 Kafka 进行交互，接收和发送消息。
- **Pandas/Numpy/Scikit-learn**：用来处理用户数据，推算用户喜好（基于历史购买记录进行数据分析和机器学习）。
- **Celery（可选）**：如果需要异步任务管理，可以用来处理 Kafka 消息的并行消费和处理。

```bash
user-preference/
├── app/
│   ├── main.py                # FastAPI应用主入口
│   ├── kafka_consumer.py      # Kafka消费者，处理接收消息
│   ├── kafka_producer.py      # Kafka生产者，发送处理结果
│   ├── services/              # 存放业务逻辑处理
│   │   ├── recommendation.py  # 推算推荐逻辑
│   ├── models/                # 数据模型（用户数据、推荐结果等）
│   ├── config.py              # 配置文件（Kafka配置等）
├── requirements.txt           # 项目依赖
├── Dockerfile                 # Docker 容器化应用文件
└── README.md                  # 项目说明文档
```


Java App 通过 kafka 将用户信息传递给 Python App ，Python App 根据用户信息 ~~【买过什么东西 ……】~~，推算出这个用户他喜欢什么产品，把这个结果再通过 Kafka 传递给 Java App
