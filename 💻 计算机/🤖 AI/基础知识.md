# Agent 智能体
Agent 是能自主完成任务的 AI 系统 —— 它不只是你问我答，而是能理解目标、制定计划、调用工具、根据反馈调整行动、最终完成任务


# LLM 大语言模型
LLM 就是 ChatGPT、Claude、Gemini 这些大模型本身，它是 Agent 的大脑

# Tool 工具
LLM 本身只会说话，不能真正执行操作，而 Tool 给了 LLM 手脚 —— 搜索网页、查数据库、发邮件、执行代码、读写文件


# MCP 协议
MCP 是 Anthropic 提出的一个开放协议/标准，用来统一 LLM 与外部工具/数据源的连接方式

MCP 是给 AI "不认识"的工具准备的说明书。但大多数常见工具，AI 已经认识了，你不需要每次都递一本 500 页的手册

### MCP 的方式 🗂️
> 每次助手来上班，你都先递给他一本 **500 页的工具手册**，说："先把这本从头到尾看完，然后再开始干活。"
> 
> 助手看了 30 分钟手册，然后用了其中 1 个工具，花 1 分钟完成任务

### CLI 的方式 🔧
> 助手来了，你说："工具间里有锤子、螺丝刀、扳手，你应该都认识，自己去拿需要的。"
> 
> 助手直接走过去拿了螺丝刀，1 分钟完成任务

# Skills
Skills 是 Agent 能做的事情的集合 —— 一组 Tool 的组合、加上特定的 Prompt / 策略、形成一个完整的能力

# RAG 检索增强生成
RAG 让 LLM 在回答问题前，先检索相关知识，然后基于检索结果生成答案

# Prompt 提示词
Prompt 就是给 AI 的指令/输入
- System Prompt：定义 AI 的角色和规则
- User Prompt：用户的具体问题
- Few-shot Examples：给几个示例让 AI 学习格式



# Context Window
Context Window 就是 LLM 一次能看到的最大文本量

# Fine-tuning 微调
Fine-tuning 就是在预训练的大模型基础上，用特定领域数据再训练，让模型在某个领域表现更好


# Embedding 向量化
把文本转换成数字向量，用于计算相似度
