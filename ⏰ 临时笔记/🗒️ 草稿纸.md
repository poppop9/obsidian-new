# 编程路线
RPC ：一种进程间的通信方式，它实现了轻松地允许一个进程调用另一个进程的服务

API 网关 ：一种设计风格和架构原则，用于创建可伸缩、轻量级、可维护和可扩展的网络服务，它承接 http 请求，调用 RPC 服务，并负责对外部请求的协议转换、参数校验、鉴权、切量、熔断、限流、监控、风控等各类共性的通用服务

单个 OrderList.tsx 文件就有 5000+ 行代码
状态管理极其复杂，包含多层嵌套的表格、表单、模态框，有大量的业务逻辑和状态相互依赖


技术架构限制
- UmiJS 的页面路由机制会完全卸载组件
- ProTable 组件有自己的状态管理逻辑
- 大量异步数据和复杂的数据流

工作量巨大 - 需要重构整个状态管理架构
风险很高 - 可能会影响现有功能的稳定性
需要深度定制 - 不是简单的通用方案能解决的



金蝶等企业级 ERP 系统是怎么做的

1. 多标签页架构
类似浏览器标签页，每个页面在独立的标签中
切换标签时页面不会卸载，只是隐藏/显示
这是最常见的企业级解决方案

2. 状态管理 + 持久化
使用 Redux/Vuex 等状态管理库
关键状态存储在全局 store 中
配合 localStorage/sessionStorage 持久化

3. Keep-Alive 机制
Vue 系统使用内置的 `<keep-alive>` 组件
React 系统使用第三方库如 react-keep-alive
