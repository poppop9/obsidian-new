## 理论
卡片盒不是存储信息的工具，它是你交流的对象，你跟卡片交流后或许可以产生新的灵感

不要纠结卡片之间是不是有严密的因果关系。只要感觉相关，就放在一起。这个系统的价值不在于"严谨分类"，而在于让相关的想法彼此靠近，日后翻到的时候能产生碰撞。

one note one idea

我觉得卡片的 idea 不需要分领域，因为有些领域的知识其实是互通的

Zettelkasten 应该是只存放思考类的笔记，可以演化、可以被引用、可以与其他想法碰撞的知识单元。非思考类的笔记不存放进来，因为它天然就是独立文档。

## 落地方案
所有的笔记都会被直接或者通过间接被引用到一个 主页面的 note 里

用 yaml 的 tags 进行领域分类

笔记中要关联，笔记末尾也要关联

Frontmatter 元数据 - 有助于搜索和查询

链接笔记一定要说明两个笔记之间的关系

```
# Decision Making Framework

## Related Concepts
- [[Confirmation Bias]] - 偏见如何影响决策
- [[Time Pressure Effects]] - 紧迫感如何改变决策质量
- [[Team Dynamics]] - 团队与个人决策的差异
```

### 原子笔记结构
标题：清晰，以名词为焦点

摘要：一句话定义
解释 / 核心思想：完整解释（费曼技巧风格）
示例：具体应用
相关 / 关联关系：链接到关联的笔记
我的思考

### MOC
MOC 是一个围绕某个主题，把相关笔记链接组织起来的导航中心

```
# 机器学习 MOC

## 基础概念
- [[监督学习]]
- [[无监督学习]]
- [[损失函数]]

## 算法
- [[线性回归]]
- [[决策树]]
- [[神经网络]]

## 实践
- [[模型评估]]
- [[特征工程]]
```

MOC 并不是知识本身，而是帮助你进入某个知识领域的导航入口

### TOC

# PARA

| 分类  | 含义        | 特点                                |
| --- | --------- | --------------------------------- |
| P   | Projects  | 有明确目标和截止时间的项目                     |
| A   | Areas     | 你必须持续维护的事情，你不维护就会出问题              |
| R   | Resources | 你感兴趣或未来可能用到的项目，你不维护也不会出事          |
| A   | Archives  | 曾经活跃，现在已完成、休眠的、失效的项目，你一般不会主动去翻的地方 |
